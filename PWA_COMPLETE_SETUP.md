### 3. **Offline Caching Strategy**

| Asset Type | Strategy | Duration | Purpose |
|-----------|----------|----------|---------|
| HTML | NetworkFirst (3600s) | 1 hour | Fresh content, fallback to cache |
| JS/CSS | CacheFirst | 30 days | Static assets, rarely change |
| Images | CacheFirst | 60 days | Game graphics and UI |
| Fonts | CacheFirst | 30 days | Typography assets |

**How it works:**
- First load: Downloads all assets
- Offline: Serves cached version
- Online: Auto-updates when changes detected

## 🔌 How to Deploy for Production

### **1. HTTPS Required**
PWA requires HTTPS in production (except localhost for development).

```bash
# For dev (HTTP is OK)
npm run dev    # http://localhost:5173

# For production (MUST be HTTPS)
npm run build  # Creates optimized dist/
```

### **2. Server Configuration**
Set these cache headers on your server:

```
# Service Worker (must revalidate)
/sw.js
  Cache-Control: public, max-age=0, must-revalidate

# Manifest (1 day cache)
/manifest.json
  Cache-Control: public, max-age=86400

# Assets (long-term cache)
/js/*, /css/*, /img/*
  Cache-Control: public, max-age=31536000, immutable

# HTML (no cache)
/index.html
  Cache-Control: public, max-age=3600, must-revalidate
```

### **3. Hosting Services (Easy Setup)**
These auto-configure PWA correctly:
- ✅ **Vercel**: Automatic (no config needed)
- ✅ **Netlify**: Automatic
- ✅ **Firebase Hosting**: Automatic
- ✅ **GitHub Pages**: Automatic with custom domain + HTTPS
- ✅ **AWS Amplify**: Automatic

### **4. Check Your Deployment**
Visit your app at `https://yourdomain.com` and:
1. Open DevTools → Application → Service Workers
2. Check if SW is registered ✅
3. Go offline (DevTools → Network → Offline)
4. Refresh page - should still work ✅
5. Look for install prompt ✅

## ❓ Troubleshooting

### **Service Worker not appearing?**
- Use HTTPS in production (HTTP only works on localhost)
- Check `dist/sw.js` exists
- Look for errors in Console tab

### **Install prompt not showing?**
- Check manifest is valid (DevTools → Application → Manifest)
- Ensure 192x192 icon exists
- Check browser console for errors
- iOS always shows "Add to Home Screen" in Share menu

### **Offline doesn't work?**
- Verify Service Worker is registered
- Check Cache Storage has entries
- Ensure assets are in `globPatterns`

### **App updates slow?**
- Service Worker caches for 30+ days
- Users must update manually or reload
- Consider faster cache expiration for testing
