# BattleZone 3D LinkedIn Posts

## Post Option 1

I am excited to share **BattleZone 3D**, a cross-platform 3D battle game I built using **React, TypeScript, Three.js, React Three Fiber, Zustand, Tailwind CSS, and Vite**.

This project was focused on building a smooth and immersive combat experience that works across **desktop, mobile, and PWA environments**. Players can move through a futuristic arena, switch between multiple weapons, fight an AI-powered enemy, reload in real time, and experience a polished HUD with responsive controls and visual feedback.

Some of the highlights:

- Real-time 3D gameplay with a third-person combat camera
- AI bot with patrol, chase, and attack behaviors
- Weapon system with rifle, sniper, and pistol loadouts
- Mobile touch controls with joystick, shoot, reload, and pause actions
- PWA support with offline readiness and install prompt
- Optimized rendering and responsive UI for better performance

To make the project production-oriented, I also prepared supporting documentation:

- **PRD** for product goals, user experience, and business value
- **FRD** for gameplay features, controls, and system behavior
- **TRD** for architecture, stack decisions, and implementation details

This project helped me strengthen my understanding of **interactive 3D experiences on the web**, performance-aware frontend development, and designing game systems with a clean React-based architecture.

## Post Option 2

Built a new project: **BattleZone 3D**

It is a browser-based 3D battle game designed to deliver a lightweight but engaging arena combat experience across devices. The goal was to combine modern frontend engineering with real-time gameplay systems in a way that feels smooth, responsive, and deployment-ready.

The project includes:

- A 3D combat arena rendered with Three.js and React Three Fiber
- State-driven gameplay flow from loading screen to lobby, battle, pause, and game over
- AI enemy behavior with movement, line-of-sight logic, and firing decisions
- Multiple weapon types with ammo, reload timing, damage values, and score rewards
- Premium HUD experience with health bars, crosshair tracking, score, and weapon selection
- Mobile-first interaction model and installable PWA support

Alongside development, I documented the project with:

- **PRD** to define product vision and player experience
- **FRD** to capture functional systems and expected behavior
- **TRD** to explain architecture, components, and technical decisions

BattleZone 3D was a great exercise in blending **frontend engineering, 3D rendering, UX, gameplay logic, and deployment thinking** into one complete product.
